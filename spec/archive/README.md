# spec/archive/

**Superseded spec documents — historical reference only.**

These are spec files that have been consolidated, renamed, or
retired in favour of a current spec. They are kept for
historical reference and to preserve the rationale that went
into the design decisions captured here, but they are **not the
source of truth for current behaviour**. For how the system
works today, see the current spec file named in each archived
doc's header note (and `docs/status.md` for shipped state).

Mirrors the convention in `guide/archive/`.
