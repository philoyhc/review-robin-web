{# Segment 13A PR 5b/5c — Inline JS for the rule editor. Walks the visible rule list on form submit and reconstructs the nested rule tree from each row's ``data-indent`` attribute, writing the JSON into the hidden ``rules_json`` field. Indent semantics: a row at indent N belongs to the nearest preceding row at indent N-1 if that preceding row is a COMPOSITE. Top-level rules have ``data-indent="0"``. Add/remove buttons work on any row; removing a composite also removes all its children (rows immediately following with greater indent). #}
