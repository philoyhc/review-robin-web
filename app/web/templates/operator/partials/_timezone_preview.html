{# Live timezone preview + deadline re-render — progressive enhancement, shared by the Create Session / Edit Session Details forms (Segment 18B PR 5) and the Operator Settings page. Requires a `display_timezone` input; the `tz-sample` / `tz-sample-zone` spans (still rendered on Operator Settings; dropped from the session forms when the timezone picker grew the GMT-prefixed datalist) and the `deadline` input / `deadline-zone` span are optional. As the operator picks a zone it updates the worked sample + zone label where present, and re-renders the deadline input so its wall-clock tracks the picked zone — the instant stays fixed, only the display moves. The no-JS fallback is the server-rendered sample + zone label. #}
