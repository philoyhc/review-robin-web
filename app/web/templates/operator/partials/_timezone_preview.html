{# Live timezone preview + deadline re-render — progressive enhancement, shared by the Create Session and Edit Session Details forms (Segment 18B PR 5). Requires a `display_timezone` input plus `tz-sample` / `tz-sample-zone` spans; the `deadline` input and `deadline-zone` span are optional. As the operator picks a zone it updates the worked sample, the CLDR zone label and the "Entered in" deadline hint, and re-renders the deadline input so its wall-clock tracks the picked zone — the instant stays fixed, only the display moves. The no-JS fallback is the server-rendered sample + the raw IANA id. #}
